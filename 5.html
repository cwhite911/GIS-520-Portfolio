Problem:

The problem presented in assignment 5 is to georeference scaled CAD data and create a geodatabase from its contents.

Return to Top


Analysis Procedures:

The strategies used to take CAD data and use it in ArcMap require the data to be georeferenced and
converted to a geodatabase. Orthophotography will be used to georeference the data and conversion tools from the ArcGIS toolbox will
be used to convert the data to more manageable types within ArcMap.

To create a geodatabase from scaled CAD data, the first step is to add the CAD drawing file and projected orthophotography to ArcMap.
From here the georeferencing tool can be used to to align the CAD drawing to features shown in the orthophotography. Once proper alignment
is made between the CAD drawing and orthophotography the georeferencing can be updated, creating a world file assoicated to the CAD drawing.
Now, a conversion tool called CAD to Geodatabase can be used to create a projected version of the CAD drawing in a file geodatabase separating
the different geometry types into a new feature dataset. Check that both the line and polyline feature classes for inconsistancies. If polygons do
not exisit where the lines represent them, use the feature to polygon tool to correct the issue. Now that the data is properly prepared, the CAD
geodatabase can be sorted as needed using the feature class to feature class tool. Querys can be defined within the tool to separate the data as needed.

Application & Reflection:

Converting CAD data to usable GIS data is commonly used while working with municipalities. Water and sewer data
is generally added to a GIS database from engineers AS-Builts. These CAD files need to be properly georeferenced
and aligned to be viable in creating complex networks. The skills and techniques used in this exercise are useful
in understanding how to take a CAD dataset and extract the useful GIS data. However, in this example the data must
be properly connected to an existing dataset. If this is not done correctly, analysis of the system would not be possible.
The data must be connected properly to reflect the mains, laterals, valves, and other associated data to the new dataset.
Also, only the relevant attribute data must be carried over to the existing dataset. When all of this is completed, the new
water system data will be properly added into the existing water geodatabase.
