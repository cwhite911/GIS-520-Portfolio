Problem:

Determine if there is a correlation between road conditions and accidents on route 30000030 in Pitt County, NC.

Analysis Procedures:

Linear referencing is the main strategy used in solving the problem. Event layers will be created for accidents and road conditions, and then the two layers will be intersected. This will provide an analytical view that will answer the question.

The methods used in this exersice (figure 1) are a basic example of linear referencing. To begin, both event data and route data must contain the proper field schema and contain a unique route field identifier. Now, the event data can be turned into an event layers by using the make route event layer tool in ArcMap. To investage how two event layers interact, the event layers must be intersected using overlay route events tool. The output feature will provide an event layer with joined event data at intersections.

Application & Reflection:

Linear referencing has many practical applications that can be used in a variety of fields. One field where linear referencing is essential is transit. Transit systems use linear referencing for multiple reasons including, tracking, reporting, or planning. A common format for delivering transit data is known as the general transit feed specification or GTFS. This is used by Google Transit and open trip planner. However, the GTFS feed is basically linear referencing only it is broken down into text files. The feed includes route information and events that capture stops, times, and bus locations, as well as the shape of the route itself. This data is used to produce very powerful and useful trip planning tool that are easy to use.
