Problem:

Create two maps that represent new job creation in both NC State Congressional Districts and NC State Senate Districts.

Return to Top


Analysis Procedures:

The strategies used to create these maps involve summarizing and joining data. Summarizing data allow joining databases to change relationships, in this case from one to many into one to one. Since the relationship is changed to one to joining tables is useful for this project. The other joining strategy is a spatial join; this allows data to be joined based off its loca not a table relationship. The data needed for this exercise can be downloaded from the library.

The methods for taking data with low cardinality and joining it to data with high cardinality starts with summarizing the data with low-cardinality, which changes the relatioinship between the datasets from a ‘one to many’ to a ‘one to one’ relationship (figure 1). The data can be summarized in various ways, but for this project the data will be sumed. Once the data is summarized it can be joined by attributes to the ZIP code points data. Once this is done, the ZIP Codes points can be spatially joined to the NC House and Senate data. In this process, the joined data’s attributes are summarized and reflected as new features class.

Application & Reflection:

Summarizing and joining data are powerful tools in GIS, which allow manipulation of data cardinality and create the ablility to compare databases based on location or tabular attributes. One example of how these techniques can be used in the field is creating a model that shows elevation profiles of a given route and scoring them based on difficulty. Routes can be spatially joined to elevation or slope data, either by the nearest feature or by assigning the average of elevation to the line feature. If someone wanted to query a route, the program could summarize the route and produce a difficulty score base on mulitutude of variables. For example, if the average slope is greater then 5 percent the score is 2 or medium. The underlying concept is how to take data of different cardinality and use them together in a productive manner.
