Assignment 13:
Elementary Customization for ArcMap

  • Problem
  • Analysis Procedures
  • Results
  • Application & Reflection

Problem:

Customize toolbars in ArcMap with non-default tools and downloaded Add-Ins. The two toolbars created were SDE and AssignmentB. The SDE tool bar includes a menu called TopoSDE that includes Topology Edit tool, Reconcile, Post and Change Version tools. Toolbar AssignmentB includes the tools Pixel Inspector, Raster Layer Property Page, Measure Raster Distance, Erase, Raster Function Template Editor, Run Easy Calculate 10 and a menu named ArcScam, which includes the Raster Painting Toolbar and Raster Snapping Options. The Interpolate Line Tool was added to the Feature Layer Context Menu.

Analysis Procedures:

The strategies needed to customize ArcMap include downloading plugins and customizing toolbars. ESRI has made it easier to install plugins by setting up new file format .esriAddIn, which is an installation wizard. Customizing the toolbar allows the user to setup their workspace for specific tasks.

To install most plugins simply run the install process by double clicking the Add-In ending in .esriAddIn. Next, open customize mode and either create a new toolbar or add the Add-In to an existing toolbar, menu or context menu. Now find the plugin in the commands tab and drag and drop it onto whichever toolbar was chosen.

Application & Reflection:

It is important to be able to customize the working environment in ArcMap and install Add-Ins because it gives the user flexibility in the methods they can use to approach problems. If a Add-In is not currently available the user can create or have someone create one for them. Customizing toolbars is important because it can help in productivity. For example, if a user is doing lots of versioned edits in SDE it would be convenient to have the versioning toolbar or their own versioned editing toolbar. An example of a good Add-In would be a simple export to geojson tool or a tool that converts map symbology to cartocss.
